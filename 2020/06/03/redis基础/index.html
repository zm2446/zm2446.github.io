什么是nosql

指的是非关系型数据库又是数据库缓存技术

NoSQL数据库的四大分类

键值(Key-Value)存储数据库

主要使用到一个哈希表，这个表中有一个特定的键和一个指针指向特定的数据。

列存储数据库

用于应对分布式存储的海量数据。

文档型数据库

键值对的升级版，特点是指向多个列

图形数据库

使用灵活的图形模型，并且能够扩展到多个服务器上。

关系型和非关系型数据库的对比

关系型数据库

优点：

易于维护：都是使用表结构，格式一样

使用方便：sql语言通用，可用于复杂查询

复杂操作：支持sql,可用于一个或多个表的复杂查询

缺点：

读写性能差，特别是海量数据的读写效率

固定的表结构，灵活度不佳

高并发读写需求

非关系型数据库：

格式灵活：可以存储任意形式的数据，应用场景广泛。

速度快：可以使用硬盘或者内存作为载体

成本低：部署简单

使用非关系型数据的原因

数据库的高并发访问

海量数据的存储

高可扩展性和高可用性

Redis的持久化

什么是持久化：

将内存中的数据写到服务器的硬盘，永久保存下来

Redis持久化的两种方式：

  1. RDB:二进制的存储格式
  2. AOF:日志记录的存储格式，文本文件

RDB

RDB持久化机制的优点：

1.方便备份与恢复

整个redis数据库将只包含一个文件，默认是dump.rdb,对于文件备份和恢复而言是非常完美的。因为我们可以非常轻松的将一个单独的文件压缩后再转移到其他存储介质上。一旦当机故障，可以很容易的进行恢复。

2.性能最大化

3.启动效率高

缺点

  1. 不能完全避免数据丢失

因为RDB是每隔一段时间写入数据，所以系统一旦在定时持久化之前出现宕机现象，此前没有来得及写入磁盘的数据都丢失。

  1. 会导致服务器暂停的现象

由于RDB是通过子进程来协助完成数据持久化工作的，因此当数据集较大时，可能会导致整个服务器停止服务几百毫秒，甚至是1秒钟。

RDB持久化的配置

在redis.windows.conf配置文件中的SNAPSHOTTING

语法 说明
save<时间间隔><修改键数> 在指定的时间后，修改了多少个键，就进行持久化的操作。
必须两个条件同时满足，修改指的是增删改

配置命令：sava 时间间隔 修改的键数

启动服务器 redis-server 配置文件名称（redis.window.conf）

AOF

优点：

包含一个格式清晰，易于理解的日志文件能用于记录所有的修改操作。也可以通过该文件能完成数据的重建。可以带来更高的安全性。所有操作都是异步完成的。

缺点：

文件比RDB更大

运行效率比RDB更慢

AOF配置

默认是关闭的，需要开启AOF模式

参数配置 说明
appendonly
no/yes		 aof存储方式的开关，设置yes表示开启，如果开启了aof，则rdb文件就失效。

AOF持久化时机

关键字 持久化时机 解释
appendfsync everysec 每秒写一次
appendfsync always 每修改写一次
appendfsync no 不写入

修改文件为yes后通过redis-server redis.windows.conf启动服务器，在服务目录下出现appendonly.aof文件。大小是0字节

打开appendonly.aof文件，查看文件的变化。会发现文件记录了所有操作的过程。

AOF重写机制

为什么需要重写

因为AOF文件越来越大，文件占用空间，而且运行速度更慢。

重写原理

通过读取服务器当前的数据库状态来实现的。

触发方式

1.手动触发：用户通过调用bgrewriteaof手动触发

2.自动触发：如果全部满足的话，就触发自动的aof重写操作

​ 1.没有RDB持久化/AOF持久化在执行，没有bgrewriteaof在进行

(删除rdb/aof文件)

​ 2.当前AOF文件大小要大于redis.conf配置的auto-aof-rewrite-min-size大小

（auto-aof-rewrite-min-size 200b 自动重写的最小尺寸）

​ 3.当前AOF文件大小和最后一次重写后的大小之间的比率等于或者等于指定的增长百分比（在配置文件设置了auto-aof-rewrite-percentage参数，不设置默认为100%）

（auto-aof-rewrite-percentage 100 大于原来的100%就自动重写）

Jedis

jedis介绍

redis不仅可以使用命令来操作，现在基本上主流的语言都有api支持，比如java,c++,node.js等。官方网站列一些java的客户端，有jedis,redission等官方推荐使用jedis和Redission.jedis操作redis需要导入jar包。

Jedis类常用方法

每个方法就是redis中的命令名，方法的参数就是命令的参数。每个jedis对象类似与JDBC中Connection对象。

常用方法

连接和关闭 功能
new Jedis(host,port) 创建一个连接
host:主机名或ip地址
port:端口号6379
void close() 使用完毕以后要关闭连接
对string操作的方法 说明
set(String key,String value) 添加键和值
String get(String key) 通过键获取值
del(String … keys) 删除一个或多个键
对hash操作的方法 说明
hset(String key,String field,String value) 添加一个hash类型的字段和值
Map<String,String> hgetall(String key) 通过键得到这个hash类型下所有的字段和值
对list操作的方法 说明
lpush(String key,String…values) 从左边添加1个或多个元素
List<String> lrange(String key,long start,long end) 获取指定范围内的元素
对set操作的方法 说明
sadd(String key,String…values) 添加一个或多个集合元素
Set<String> smembers(String key) 通过键获取集合中所有的元素

jedis的连接池作用

反复的创建和销毁jedis很消耗程序性能

API

配置类 说明
new JedisPoolCofig() 创建连接池的配置对象
void setMaxTotal() 设置最大的连接数
void setMaxWaitMillis() 设置获取连接最长的等待时间
连接池类 说明
JedisPool(配置对象，服务器名，端口号) 创建连接池对象
Jedis getResource() 从连接池中得到一个连接对象

代码：

JedisPoolConfig config = new JedisPoolConfig();

config.setMaxTotal(10) 设置最大的连接数

config.setMaxWaitMillis() 设置获取连接最长的等待时间

JedisPool pool = new JedisPool(config,”localhost”,6379);

Jedis jedis = pool.getResource(); 获取连接对象

jedis,close() 关闭连接对象

ResourceBundle:

public static ResourceBundle getBundle(“配置文件基名”) 从src下读取这个属性文件

String getString(“键名”) 通过键得到值
